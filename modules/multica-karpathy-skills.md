# Source: multica-ai/andrej-karpathy-skills — CLAUDE.md

> Original: https://github.com/multica-ai/andrej-karpathy-skills/blob/main/CLAUDE.md

A single behavioral CLAUDE.md derived from Andrej Karpathy's observations on LLM
coding pitfalls. Notable for being short and biasing "toward caution over speed"
while explicitly carving out trivial tasks for judgment.

Key contributions to draft/CLAUDE.md:
- "Think before coding" (state assumptions, don't hide confusion, surface tradeoffs)
- Simplicity First (minimum code, nothing speculative — folded into "no speculative abstraction")
- Surgical Changes (touch only what you must; every changed line traces to the request; leave pre-existing dead code alone)
- Goal-Driven Execution (turn tasks into verifiable goals: write the failing test, then make it pass)
- The framing that guidelines should bias toward caution but defer to judgment on trivial work
